# 最终目标效果（grilling 定稿）

> 通过 grilling 逐题拷问锁定。术语见 [CONTEXT.md](../CONTEXT.md)，关键决策见 [docs/adr](./adr)。

## 一句话

国服海克斯大乱斗玩家的桌面助手：**游戏内浮层实时看「这局这英雄该优先抢哪些海克斯/走哪套三连」**（北极星），辅以选英雄时客户端一键导入出装、离线速查库。

## 三个交付物

### ① 游戏内浮层（北极星）
- 常驻 OS 置顶半透明小窗 + 热键收起/呼出。
- **自动识别本局英雄** → 显示**完整决策卡**：海克斯优先级排序（按胜率，分白银/黄金/棱彩）+ top 几个三连组合 + 核心出装。
- 前提：游戏设为**无边框**模式（置顶窗盖不住独占全屏）。
- **不做**：不替你选三选一（官方 API 拿不到游戏内实际给出的 3 个选项），只给排序供对照。

### ② 客户端选英雄出装助手
- 选/切英雄时**停靠客户端右侧的置顶窗**，自动识别英雄。
- 展示海克斯大乱斗出装+符文 → **一键导入**「预设出装」(item-set) + 符文页到客户端。
- 符文导入 Nidalee 已有；需补 item-set 写入 + 停靠面板。

### ③ 速查库
- 复用①②的数据，加搜索/浏览入口：不打游戏时查任意英雄的海克斯优先级/出装/三连 + 全局海克斯 tier 表（可搜）。不额外造数据。

## 智能 = 数据驱动，不是标签驱动

- v1 对局感知只绑「你本局英雄」，拉其真实战绩推荐。
- 阵容/敌我维度**不做静态 AP/AD 标签启发式**——海克斯打乱定位，标签在此模式不可靠（见 CONTEXT「静态标签失效」）。要做也须数据支撑，留 v2。
- 动态补齐（按已选海克斯反查三连推荐下一个）数据支持但复杂，留 v2。

## 数据

- 主源 Blitz Datalake（外服 KR），UI 标注来源/版本（见 ADR-0001）。
- 双外服源容灾 + 深探海斗小助手 = v1 后并行。

## 平台

- 玩：国服 + Windows。构建目标 Windows（Mac 仅开发机）。

## 明确不做（红线）

- 不注入游戏内存 / 不读屏 / 不 hook DirectX。
- 不自动识别并替选游戏内三选一。
- 不伪造国服精准数据（外服代理如实标注）。
