# 浮层用 OS 置顶窗，不做游戏内注入

游戏内海克斯优先级提示用操作系统级置顶窗实现（Tauri always-on-top，=Blitz/Porofessor/OP.GG 同款），不做 DirectX 注入/hook 把提示画进游戏画面。注入方案体验略好但重、跨版本脆、贴近 Riot 不鼓励的读屏区域；置顶窗稳、零封号风险、原生支持。

## Consequences

- **硬约束**：OS 置顶窗盖不住独占全屏——游戏必须设为**无边框(Borderless)**模式浮层才可见。这是物理约束不是选择；现代机上无边框 vs 全屏性能差异可忽略。
- 浮层形态：常驻置顶半透明小窗（自动跟随本局英雄）+ 热键收起/呼出。
- 官方 API 拿不到游戏内实际给出的 3 个海克斯选项，故浮层只做「该英雄优先级排序」对照，不做「替你选三选一」。
