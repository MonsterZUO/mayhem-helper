# 海克斯大乱斗助手 (Mayhem Helper)

国服海克斯大乱斗（ARAM: Mayhem）的桌面辅助工具：游戏内浮层看海克斯优先级、客户端选英雄时一键导入出装、离线速查库。基于 Nidalee（Tauri）基座。

## Language

### 模式与玩法

**海克斯大乱斗 (ARAM: Mayhem)**:
嚎哭深渊随机英雄 + 海克斯增强的模式。本项目默认且主要服务此模式。
_Avoid_: 极地大乱斗（指普通 ARAM，无海克斯）、斗魂竞技场（指 Arena 2v2v2v2）

**海克斯 (Augment)**:
局内在等级 7/11/15 三选一获得的强化。三档稀有度：白银 / 黄金 / 棱彩(Prismatic)。**每次三选一给出的三个选项为同一档位**——故任何推荐/展示都应支持档内比较（UI 三列并排的依据）。
_Avoid_: 强化符文（arammayhem 的叫法，会与「符文/rune」撞车）、增强

**符文 (Rune / Perk)**:
英雄联盟常规天赋符文页（LCU `/lol-perks`），与海克斯是两回事。
_Avoid_: 强化符文

**三连组合 (Augment Trio)**:
一局三次选择凑成的三海克斯组合，是「套路」的最小单位，有整体胜率。
_Avoid_: 套路（太泛）、combo

### 推荐

**优先级排序 (Priority Ranking)**:
因无法得知游戏内实际给出的那 3 个选项（官方 API 不暴露），推荐形态是「该英雄全部海克斯按胜率排序」，玩家拿它对照游戏里出现的选项。**不是**替玩家选三选一。

**对局感知 (Match-aware)**:
把推荐绑定到当前这一局。v1 仅绑定「你本局的英雄」（LCU 可靠获取）。

**静态标签失效 (Archetype Breakdown)**:
海克斯大乱斗特有现象——海克斯颠覆英雄静态定位（AP 英雄可能出 AD，出装无定式）。推论：推荐必须**数据驱动**（真实出装/海克斯胜率），不能靠「AP/AD/坦克」身份标签做启发式。此原则也约束未来的阵容维度功能。

### 客户端交互面

**预设出装 (Item Set)**:
可写入英雄联盟客户端的自定义出装（LCU `/lol-item-sets`），局内商店可见。
_Avoid_: 出装方案、build

**出装助手面板 (Build Panel)**:
选/切英雄时停靠客户端右侧的置顶窗，展示出装+符文并提供一键导入。

**浮层 (Overlay)**:
游戏运行时的 OS 置顶窗（非注入、非读屏），显示本局英雄的海克斯优先级。需游戏设为无边框模式方可覆盖显示。

### 数据

**Blitz Datalake**:
主数据源（`datalake.v2.iesdev.com/graphql`），提供海克斯大乱斗真实战绩：海克斯/出装/三连组合的胜率、选取率、tier。查询只吃 champion_id、无地区参数。

**外服代理数据 (KR-proxy Data)**:
关键约束——市面所有可取源（Blitz/arammayhem/aramkit/aramgg）全是外服(KR/全球)，**真国服统计数据不存在**（Riot API 不暴露、腾讯国服无开放接口）。故用户虽玩国服，用外服数据作代理，UI 必须标注「来源 KR / 版本 X」。相对优先级排序由机制驱动、跨服可迁移，是代理成立的依据。
